# 🛠️ TINY Language Compiler

A robust compiler implementation for the extended TINY programming language, written entirely in C#. This project is developed as part of the Compiler Construction coursework for the Computer Engineering program at Ain Shams University.

## 📖 About the TINY Language
TINY is a pedagogical programming language designed for learning compiler architecture. This specific implementation supports an extended 30-rule grammar, featuring:
* **Data Types:** `int`, `float`, `string`
* **Control Structures:** `if-elseif-else-end`, `repeat-until`
* **Functions:** Full support for function declarations, parameters, and a required `main()` entry point.
* **Operators:** Standard arithmetic, assignment (`:=`), conditional, and boolean operators.

## 🏗️ Compiler Architecture

The compiler is being built in distinct phases, taking raw source code and translating it through a standard compiler pipeline.

### Phase 1: Lexical Analysis (Scanner) — ✅ *Completed*
The Scanner acts as the first line of processing. It reads the raw TINY source code character by character and groups them into meaningful, categorized `Tokens`.
* **Engine:** Custom-built deterministic state machine using pre-compiled Regular Expressions (.NET flavor).
* **Features:**
  * Accurate line tracking for multi-line comments and whitespace.
  * O(1) keyword validation.
  * Robust error handling (generates `Unknown` tokens for illegal characters to prevent crashing).
  * Injects an `EOF` (End of File) token for parser safety.

### Phase 2: Syntax Analysis (Parser) — 🚧 *Planned*
The Parser will consume the token stream generated by the Scanner and verify it against the TINY Context-Free Grammar (CFG).
* Will generate an Abstract Syntax Tree (AST).
* Will catch and report structural syntax errors (e.g., "Missing ';' on Line 15").

### Phase 3: Semantic Analysis & Code Generation — 📅 *Upcoming*
* Type checking and scope validation (Symbol Table).
* Translating the AST into intermediate representation or target machine code.
